# 后端代码阅读指南

> 面向前端开发者的后端代码学习路径，帮助你系统地理解这个健身训练记录微信小程序的后端服务。

## 项目概况

- **框架**: Express 5 (JavaScript, 无 TypeScript)
- **数据库**: MySQL + Prisma ORM
- **认证**: JWT (JSON Web Token)
- **架构**: 单体应用，路由即控制器，无独立 Service 层
- **核心代码量**: 约 1500 行，结构清晰，适合入门

## 学习路径

### 第一步：看入口，理解请求生命周期

**阅读文件**: `src/app.js`

理解一个 HTTP 请求从进入到响应经过了哪些环节：

```
请求进入 → helmet(安全头) → cors(跨域) → morgan(日志) → JSON解析 → 限流 → 路由匹配 → 错误处理 → 响应返回
```

**关注点**：
- 中间件的挂载顺序（顺序很重要）
- 公开路由和受保护路由的区分方式
- 路由前缀的挂载规则（如 `/api/auth`、`/api/workouts`）

### 第二步：看中间件，理解横切逻辑

**阅读目录**: `src/middleware/`

按以下顺序阅读：

| 顺序 | 文件 | 核心概念 |
|------|------|---------|
| 1 | `auth.js` | JWT 鉴权，理解 token 验证机制（对应前端 localStorage 里存的 token） |
| 2 | `validate.js` | 参数校验，对应前端表单校验的后端版本 |
| 3 | `errorHandler.js` | 全局错误处理，理解不同错误码（401/404/409/500）的含义 |

### 第三步：看路由，理解 CRUD 模式

**阅读目录**: `src/routes/`

按复杂度从低到高阅读：

| 顺序 | 文件 | 复杂度 | 学习重点 |
|------|------|--------|---------|
| 1 | `auth.js` | 低 | 微信登录流程、JWT 签发、密码哈希比对 |
| 2 | `checkin.js` | 低 | 最简单的业务逻辑，只有查询和创建 |
| 3 | `user.js` | 低 | 用户信息的读取和更新 |
| 4 | `exercise.js` | 低 | 纯查询路由，理解分类筛选 |
| 5 | `exercises.js` | 中 | 标准 CRUD 模板（增删改查），含分页和软删除 |
| 6 | `template.js` | 中 | 模板管理，涉及关联表操作 |
| 7 | `dashboard.js` | 中 | 聚合查询，多个数据源合并返回 |
| 8 | `stats.js` | 中 | 统计查询，日期范围和分组聚合 |
| 9 | `workout.js` | 高 | 训练记录，涉及多表关联、事务操作，复杂度最高 |

### 第四步：看数据库设计，理解数据建模

**阅读文件**：

| 顺序 | 文件 | 学习重点 |
|------|------|---------|
| 1 | `docx/后端接口与数据库设计.md` | 项目自带的设计文档，理解整体设计思路 |
| 2 | `prisma/schema.prisma` | 数据模型定义，表字段和表间关系 |
| 3 | `prisma/migrations/` | 数据库迁移 SQL，理解表结构演变过程 |
| 4 | `scripts/seedBaseData.js` | 种子数据初始化脚本，理解基础数据结构 |

**核心数据关系**：

```
User (用户)
 ├── 1:N → WorkoutSession (训练会话)
 │          ├── 1:N → WorkoutExercise (训练动作)
 │          │          └── 1:N → WorkoutSet (训练组)
 │          └── 快照机制: 记录训练时的动作信息
 ├── 1:N → WorkoutTemplate (训练模板)
 │          └── 1:N → WorkoutTemplateItem (模板动作)
 ├── 1:N → Checkin (打卡记录)
 └── 1:N → Exercise (自定义动作, isCustom=true)

ExerciseCategory (动作分类: 胸/背/肩/腿/手臂)
 └── 1:N → ExerciseGroup (动作分区: 上胸/中胸/背阔肌...)
              └── 1:N → Exercise (动作库)
```

### 第五步：看工具函数，理解通用封装

**阅读目录**: `src/utils/`

| 文件 | 作用 | 前端对应概念 |
|------|------|-------------|
| `response.js` | 统一响应格式 `{ code, message, data }` | 前端 axios 拦截器里解析的结构 |
| `prisma.js` | 数据库连接单例 | 类似前端的 axios 实例 |
| `wechat.js` | 调用微信 `code2session` API | 类似前端调第三方 SDK |

## 前后端概念对照表

| 你在前端做的事 | 对应后端的什么 |
|---|---|
| `axios.post('/api/auth/login', data)` | `routes/auth.js` 里的 `router.post('/login', ...)` |
| 请求头带 `Authorization: Bearer xxx` | `middleware/auth.js` 解析并验证 token |
| 接口返回 `{ code: 0, data, message }` | `utils/response.js` 的 `success()` / `fail()` 封装 |
| 表单校验（如必填、长度限制） | `express-validator` + `middleware/validate.js` |
| 接口报 401 跳登录页 | `middleware/auth.js` 验证失败 / `errorHandler.js` 捕获 JWT 错误 |
| 接口报 500 显示错误提示 | `middleware/errorHandler.js` 捕获未处理异常 |
| 列表分页 `?page=1&pageSize=10` | 路由里用 Prisma 的 `skip` / `take` 查询 |
| 前端状态管理 (store) | 数据库表 (通过 Prisma ORM 操作) |
| 前端路由 (vue-router / react-router) | Express 路由 (`express.Router()`) |
| 前端拦截器 (axios interceptors) | Express 中间件 (middleware) |

## 实操建议

### 1. 先把项目跑起来

```bash
# 安装依赖
npm install

# 复制环境变量配置（需要自行填写数据库连接等信息）
cp .env.example .env

# 执行数据库迁移
npm run db:migrate

# 生成 Prisma Client
npm run db:generate

# 初始化种子数据
node scripts/seedBaseData.js

# 启动开发服务
npm run dev
```

### 2. 用工具调接口

用 Postman、Apifox 或 curl 手动调用接口，观察请求和响应的完整结构。建议从以下接口开始：

1. `POST /api/auth/login` — 登录拿 token
2. `GET /api/exercise/categories` — 带 token 查询分类
3. `GET /api/exercises` — 带 token 查询动作列表

### 3. 加日志辅助理解

在路由 handler 里加 `console.log`，观察数据在各层之间的流转：

```javascript
// 例如在 routes/exercises.js 的查询接口中
router.get('/', authMiddleware, async (req, res, next) => {
  console.log('收到请求，查询参数:', req.query);
  console.log('当前用户ID:', req.userId);
  // ... 原有逻辑
});
```

### 4. 尝试修改代码

走通"改 schema → 迁移 → 改路由"的完整流程：

1. 在 `prisma/schema.prisma` 中给某个模型加个字段
2. 运行 `npm run db:migrate` 生成迁移
3. 在对应路由中读取/写入新字段
4. 用 Postman 验证

### 5. 前后端对照阅读

如果你有对应的微信小程序前端代码，找到前端调用某个接口的地方，再去后端路由里看对应的处理逻辑，前后端对照理解效果最好。
