# SurvivalPracticeApp 后端接口与数据库设计（MySQL）

> 说明：本设计基于当前前端页面（已排除徽章墙相关页面与数据）。适配 UniApp 前端现有的登录、动作库、训练选择/记录、训练总结、我的页面与首页概览。

---

## 1. 设计目标
- 满足当前前端页面的数据需求（不包含徽章墙）。
- 统一接口返回结构，适配前端 `ResultEnum`（`code` 为 `0` 或 `200` 表示成功）。
- 便于后续扩展：训练模板、统计、动作库自定义。

---

## 2. 通用约定
### 2.1 接口基础
- Base URL: `/api`
- 认证：JWT（`Authorization: Bearer <token>`）
- 时间：统一使用 `UTC` 或 `Asia/Shanghai`，字段为 `DATETIME`/`TIMESTAMP`

### 2.2 统一响应
```json
{
  "code": 0,
  "msg": "ok",
  "data": {}
}
```
- `code = 0 | 200` 为成功
- `code != 0/200` 统一视为业务失败

### 2.3 分页结构
```json
{
  "list": [],
  "page": 1,
  "pageSize": 10,
  "total": 0
}
```

## 3. API 设计

### 3.1 认证与用户
#### 3.1.1 微信/账号登录
`POST /auth/login`
- 微信登录：`{ "code": "wx_code" }`
- 账号登录：`{ "username": "xx", "password": "yy" }`

**响应**
```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "token": "jwt_token",
    "userInfo": {
      "id": 1,
      "nickname": "微信用户",
      "avatar": "/static/images/default-avatar.png"
    }
  }
}
```

#### 3.1.2 获取用户信息
`GET /user/info`

#### 3.1.3 退出登录
`GET /auth/logout`

## 4. 数据库表设计（MySQL 8）

### 4.1 用户表
```sql
CREATE TABLE `users` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `wx_openid` VARCHAR(64) DEFAULT NULL,
  `unionid` VARCHAR(64) DEFAULT NULL,
  `username` VARCHAR(64) DEFAULT NULL,
  `password_hash` VARCHAR(255) DEFAULT NULL,
  `nickname` VARCHAR(64) NOT NULL DEFAULT '健身达人',
  `avatar` VARCHAR(255) DEFAULT NULL,
  `status` TINYINT NOT NULL DEFAULT 1,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY `uniq_openid` (`wx_openid`),
  UNIQUE KEY `uniq_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.2 Token 表（可选）
```sql
CREATE TABLE `auth_tokens` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `token` VARCHAR(512) NOT NULL,
  `expires_at` DATETIME NOT NULL,
  `revoked_at` DATETIME DEFAULT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  KEY `idx_user` (`user_id`),
  CONSTRAINT `fk_auth_user` FOREIGN KEY (`user_id`) REFERENCES `users`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.3 动作分类
```sql
CREATE TABLE `exercise_categories` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `code` VARCHAR(32) NOT NULL,
  `name` VARCHAR(64) NOT NULL,
  `sort` INT NOT NULL DEFAULT 0,
  `color` VARCHAR(16) DEFAULT NULL,
  UNIQUE KEY `uniq_code` (`code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.4 动作分区
```sql
CREATE TABLE `exercise_groups` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `category_id` BIGINT NOT NULL,
  `code` VARCHAR(32) NOT NULL,
  `name` VARCHAR(64) NOT NULL,
  `sort` INT NOT NULL DEFAULT 0,
  `description` VARCHAR(255) DEFAULT NULL,
  UNIQUE KEY `uniq_group` (`category_id`, `code`),
  KEY `idx_category` (`category_id`),
  CONSTRAINT `fk_group_category` FOREIGN KEY (`category_id`) REFERENCES `exercise_categories`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.5 动作库
```sql
CREATE TABLE `exercises` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `category_id` BIGINT NOT NULL,
  `group_id` BIGINT NOT NULL,
  `name` VARCHAR(128) NOT NULL,
  `equipment` VARCHAR(64) DEFAULT NULL,
  `is_custom` TINYINT NOT NULL DEFAULT 0,
  `created_by` BIGINT DEFAULT NULL,
  `is_active` TINYINT NOT NULL DEFAULT 1,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY `idx_category` (`category_id`),
  KEY `idx_group` (`group_id`),
  KEY `idx_creator` (`created_by`),
  CONSTRAINT `fk_ex_category` FOREIGN KEY (`category_id`) REFERENCES `exercise_categories`(`id`),
  CONSTRAINT `fk_ex_group` FOREIGN KEY (`group_id`) REFERENCES `exercise_groups`(`id`),
  CONSTRAINT `fk_ex_user` FOREIGN KEY (`created_by`) REFERENCES `users`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.6 训练会话
```sql
CREATE TABLE `workout_sessions` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `start_time` DATETIME NOT NULL,
  `end_time` DATETIME DEFAULT NULL,
  `duration_sec` INT DEFAULT NULL,
  `total_volume` INT DEFAULT 0,
  `total_sets` INT DEFAULT 0,
  `completed_sets` INT DEFAULT 0,
  `calories_est` INT DEFAULT 0,
  `status` TINYINT NOT NULL DEFAULT 1,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  KEY `idx_user_time` (`user_id`, `start_time`),
  CONSTRAINT `fk_workout_user` FOREIGN KEY (`user_id`) REFERENCES `users`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.7 训练动作（快照）
```sql
CREATE TABLE `workout_exercises` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `workout_id` BIGINT NOT NULL,
  `exercise_id` BIGINT DEFAULT NULL,
  `name_snapshot` VARCHAR(128) NOT NULL,
  `category_snapshot` VARCHAR(64) NOT NULL,
  `group_snapshot` VARCHAR(64) DEFAULT NULL,
  `sort` INT NOT NULL DEFAULT 0,
  KEY `idx_workout` (`workout_id`),
  CONSTRAINT `fk_we_workout` FOREIGN KEY (`workout_id`) REFERENCES `workout_sessions`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.8 训练组
```sql
CREATE TABLE `workout_sets` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `workout_exercise_id` BIGINT NOT NULL,
  `set_index` INT NOT NULL,
  `weight` DECIMAL(8,2) DEFAULT NULL,
  `reps` INT DEFAULT NULL,
  `completed` TINYINT NOT NULL DEFAULT 0,
  KEY `idx_workout_ex` (`workout_exercise_id`),
  CONSTRAINT `fk_ws_ex` FOREIGN KEY (`workout_exercise_id`) REFERENCES `workout_exercises`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.9 打卡记录
```sql
CREATE TABLE `checkins` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `checkin_date` DATE NOT NULL,
  `checkin_time` TIME DEFAULT NULL,
  `location` VARCHAR(128) DEFAULT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uniq_user_date` (`user_id`, `checkin_date`),
  CONSTRAINT `fk_checkin_user` FOREIGN KEY (`user_id`) REFERENCES `users`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.10 模板
```sql
CREATE TABLE `workout_templates` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `name` VARCHAR(64) NOT NULL,
  `icon` VARCHAR(16) DEFAULT NULL,
  `description` VARCHAR(255) DEFAULT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY `idx_user` (`user_id`),
  CONSTRAINT `fk_template_user` FOREIGN KEY (`user_id`) REFERENCES `users`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.11 模板动作
```sql
CREATE TABLE `workout_template_items` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `template_id` BIGINT NOT NULL,
  `exercise_id` BIGINT NOT NULL,
  `sort` INT NOT NULL DEFAULT 0,
  KEY `idx_template` (`template_id`),
  CONSTRAINT `fk_ti_template` FOREIGN KEY (`template_id`) REFERENCES `workout_templates`(`id`),
  CONSTRAINT `fk_ti_exercise` FOREIGN KEY (`exercise_id`) REFERENCES `exercises`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 5. 与前端页面的数据映射
- 首页 `pages/index/index`：`/dashboard/home` + `/checkins/status`
- 登录 `pages/login/index`：`/auth/login`
- 我的 `pages/me/me`：`/user/info` + `/auth/logout`
- 动作库 `pages/exercise/*`：`/exercise/categories`、`/exercise/groups`、`/exercises`
- 训练记录 `pages/workout/recording`：`/workouts`、`/workouts/:id/finish`
- 训练总结 `pages/workout/summary`：`/workouts/latest` 或 `GET /workouts/:id`
- 训练页 `pages/workout/index`：`/templates`、`/stats/weekly`
