## 3. 设计基准（ui-ux-pro-max 取向 + 项目主题）

根据 ui-ux-pro-max 的设计系统结果（运动/健身向）：

- **风格取向**：Vibrant & Block-based（活力、几何块面、对比鲜明）
- **氛围关键词**：Energetic / Modern / Athletic
- **强调色建议**：能量橙 + 绿色（用于成功/行动）

结合项目现有深色科技感 UI（`#0b0f1a` + `#2ee6a6` + `#f6b44d`），最终反馈体系采用：


| 角色    | 色值                     | 用途           |
| ------- | ------------------------ | -------------- |
| Success | `#2EE6A6`                | 成功 / 完成    |
| Info    | `#7CF7D4`                | 提示 / 信息    |
| Warning | `#F6B44D`                | 警告 / 注意    |
| Error   | `#FF6B6B`                | 失败 / 错误    |
| Surface | `rgba(20, 26, 40, 0.92)` | Toast 卡片背景 |
| Border  | `rgba(255,255,255,0.12)` | 玻璃卡片边框   |

字体建议保持当前（系统字体），标题加粗，正文中等；强调“清晰 + 简洁 + 轻动效”。

## 4. 反馈体系改造方案

### 4.1 全局 Toast（统一替换 uni.showToast）

**设计目标：** 轻量、克制、玻璃质感、统一位置、统一动效。

**样式规格**

- 位置：底部居中（距安全区上方 24rpx）
- 尺寸：`max-width: 82vw`，`min-height: 72rpx`
- 布局：`icon + 文案` 横向排列，间距 16rpx
- 背景：`Surface` + 轻量渐变（透明度 0.88）
- 边框：`1px solid Border`
- 阴影：`0 18rpx 40rpx rgba(0,0,0,0.35)`

**动效**

- 进场：`translateY(12rpx) + fadeIn`（200ms）
- 出场：`translateY(8rpx) + fadeOut`（180ms）
- 自动消失：2.2s（成功/提示），3.2s（错误）

**状态色与图标**

- Success：绿色描边 + 勾选线性图标（非 emoji）
- Warning：橙色描边 + 提示图标
- Error：红色描边 + 叉号图标
- Info：青色描边 + 信息图标

**使用场景**

- 动作库新增/删除/错误提示
- 训练开始 / 保存失败
- 登录提示、修改资料提示
- 全局请求错误

### 4.2 打卡反馈（替代“全屏成功弹层”）

**设计目标：** “完成感”明显，但避免遮挡与过度打断。

**方案 A（推荐）**：卡片内 Success 状态 + Toast 组合

- 打卡按钮完成后变为「已完成」状态（带微光描边）
- 在打卡卡片顶部出现一条轻量成功提示条（2s 后淡出）
- 同时触发全局 Toast（成功提示）

**方案 B（保留沉浸式）**：轻量成功浮层（非全屏遮挡）

- 浮层高度约 42% 屏幕，底部上滑浮层
- 渐变背景 + 勾选图标 + 「打卡成功」主标题 + 连续打卡信息
- 1.2s 自动收起，不遮挡整个页面

> 推荐 A，符合“快速反馈 + 不打断阅读”原则。

### 4.3 打卡 Loading（替代系统 showLoading）

**设计目标：** 保留等待感，避免系统弹窗突兀。

- 优先使用按钮内局部 loading（按钮文案变为“打卡中…” + 小型 spinner）
- 当请求超过 800ms 时，启用半透明卡片 loading（复用 `fg-loading` 视觉）
- 关闭时与成功提示平滑衔接（避免闪烁）

### 4.4 动作库新增/删除反馈

**设计目标：** 操作反馈轻而快，避免重复遮挡。

- 使用全局 Toast
- 删除动作时可在该条目上短暂高亮（轻微绿/红底）
- 表单校验类提示统一为 Info Toast（2.2s）
