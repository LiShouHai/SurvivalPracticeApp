# uni-app 多主题方案设计（Light / Dark）

## 一、目标

- 支持 **明亮（Light）/ 暗黑（Dark）** 两套主题
- 支持：
  - 手动切换
  - 跟随系统
  - 持久化存储
- 适配端：
  - H5
  - App
  - 小程序
- 最小侵入业务代码

------

## 二、推荐最优方案

### ✅ 方案结论：**CSS 变量 + 根节点 class 切换**

这是当前 Vue3 + uni-app 最稳定、维护成本最低、扩展性最好的方案。

核心思想：

```
通过 <html> 或 <body> 上的 class 控制主题
通过 CSS 变量定义不同主题色
组件内部只使用变量，不写具体颜色
```

------

## 三、目录结构建议

```
/styles
  ├── theme.scss
  ├── light.scss
  ├── dark.scss
/store
  ├── theme.ts
```

------

## 四、主题定义方式

### 1️⃣ 定义基础变量（theme.scss）

```
:root {
  --color-primary: #2979ff;
  --color-bg: #ffffff;
  --color-text: #333333;
}
```

------

### 2️⃣ 明亮主题（light.scss）

```
.theme-light {
  --color-primary: #2979ff;
  --color-bg: #ffffff;
  --color-text: #333333;
}
```

------

### 3️⃣ 暗黑主题（dark.scss）

```
.theme-dark {
  --color-primary: #5c8dff;
  --color-bg: #121212;
  --color-text: #e5e5e5;
}
```

------

## 五、组件使用方式

所有组件统一使用变量：

```
.container {
  background: var(--color-bg);
  color: var(--color-text);
}
```

⚠ 禁止写死颜色值。

------

## 六、主题切换实现

### 1️⃣ 创建 theme store（Pinia）

```
import { defineStore } from 'pinia'

export const useThemeStore = defineStore('theme', {
  state: () => ({
    mode: 'light' // light | dark | system
  }),
  actions: {
    setTheme(mode: 'light' | 'dark') {
      this.mode = mode
      uni.setStorageSync('theme', mode)
      this.applyTheme()
    },
    applyTheme() {
      const className = this.mode === 'dark' ? 'theme-dark' : 'theme-light'
      document.documentElement.className = className
    }
  }
})
```

------

### 2️⃣ App.vue 初始化

```
onLaunch(() => {
  const theme = uni.getStorageSync('theme') || 'light'
  themeStore.setTheme(theme)
})
```

------

## 七、支持跟随系统

### H5

```
const media = window.matchMedia('(prefers-color-scheme: dark)')
if (media.matches) {
  setTheme('dark')
}
```

监听变化：

```
media.addEventListener('change', e => {
  setTheme(e.matches ? 'dark' : 'light')
})
```

------

### 小程序

使用：

```
"darkmode": true
```

并监听：

```
wx.onThemeChange(res => {
  setTheme(res.theme)
})
```

## 八、跨端注意事项

| 端         | 是否支持 CSS 变量 |
| ---------- | ----------------- |
| H5         | ✅                 |
| App        | ✅                 |
| 微信小程序 | ✅（基础库 2.10+） |

如果必须兼容老小程序版本 → 使用 SCSS 变量 + 条件编译，但不推荐。

------

## 九、为什么这是最优方案？

| 方案            | 优点                       | 缺点           |
| --------------- | -------------------------- | -------------- |
| CSS 变量        | 性能好、实时切换、代码干净 | 需统一变量体系 |
| SCSS 编译多份   | 实现简单                   | 无法运行时切换 |
| 动态 style 绑定 | 灵活                       | 维护灾难       |

结论：

> 中大型项目必须走 CSS 变量体系。
