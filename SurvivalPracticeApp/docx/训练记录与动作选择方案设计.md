# 训练记录与动作选择功能设计方案

## 1. 业务流程 (User Journey) - 细化版

为了实现跨部位（如练胸+三头）的顺畅选择，我们需要建立一个“全局暂存区”的概念。

### 详细操作步骤：
1. **训练启动**：用户在“训练”频道点击【开始挑选动作】。
2. **进入分类页**：跳转至 `pages/exercise/index`（带参数 `mode=select`），用户看到胸、背、肩、腿等大类。
3. **进入具体部位**：
    - 用户点击“胸部”，进入 `pages/exercise/chest/index`。
    - **多选操作**：页面卡片显示勾选框，用户勾选“平板杠铃卧推”、“上斜哑铃飞鸟”。
    - **实时反馈**：屏幕底部出现悬浮工具条（Selection Bar），显示“已选 2 个动作”。
4. **跨部位挑选**：
    - 用户点击左上角“返回”回到分类页（此时悬浮条依然存在，记录已选的 2 个动作）。
    - 用户进入“三头”部位，勾选“绳索下压”。
    - 底部悬浮条更新为“已选 3 个动作”。
5. **确认并预览**：
    - 用户点击悬浮条上的【预览/查看】按钮，弹出一个半屏列表，展示已选的 3 个动作（可在此处调整顺序或删除）。
6. **正式开始**：
    - 点击悬浮条上的【开始训练】，正式进入 `pages/workout/recording` 记录页面。

## 2. 页面交互设计

### A. 动作分类页与列表页 (Selection Mode)
- **状态持久化**：使用全局 Store（Pinia）存储 `selectedExercises` 数组。这样无论用户在部位页面间如何切换，选择都不会丢失。
- **UI 变化**：
    - 当 `mode=select` 时，动作卡片右侧显示圆形勾选框（Checkbox）。
    - 整个 App 底部（或特定路由下）显示一个全局浮动动作条。

### B. 全局悬浮工具条 (GlobalSelectionBar)
- **展示信息**：已选数量。
- **功能按钮**：
    - 【清空】：一键重置已选。
    - 【清单】：展开查看已选详情。
    - 【开始训练】：高亮按钮，点击后清除选择状态并初始化记录页。

### C. 记录页面 (Recording Page)
- **动作卡片化**：每个动作是一个独立卡片，防止数据混淆。
- **智能填充**：默认显示第一组，并自动拉取该动作的“历史记录”或“上次训练重量”作为默认值（灰色占位符）。
- **操作逻辑**：
    - 点击“重量/次数”：直接激活输入，并支持快捷加减（如 +2.5kg, +5kg）。
    - 勾选“完成”：该组背景变色（如绿色），表示已记录。

### B. 关键组件开发
1. **`SetRow` 组件**：单组数据的展示与输入行。
2. **`WorkoutExerciseCard` 组件**：单个动作的容器，包含该动作下的所有组。
3. **`FloatingSelectionBar` 组件**：在挑选动作时底部的悬浮确认栏。

## 4. 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
| :--- | :--- | :--- | :--- |
| **独立页面** | 沉浸感强，适合复杂输入，防止误触返回导致数据丢失，易于扩展计时器等功能。 | 需要页面跳转。 | ⭐⭐⭐⭐⭐ |
| **底部弹窗 (ActionSheet)** | 交互轻快，不离开当前上下文。 | 记录多个动作时屏幕空间不足，难以承载大量的重量/次数输入。 | ⭐⭐ |
| **侧边抽屉** | 方便切换动作库和记录。 | 屏幕利用率低，不适合作为训练中的核心操作区。 | ⭐⭐⭐ |
